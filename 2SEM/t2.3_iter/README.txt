ЦЫБУЛИН ЕГОР 108 группа (2025 год)


Условие: Решить систему линейных уравнений:
1. методом простой итерации;
2. итерационным методом скорейшего спуска;
3. итерационным методом Зейделя.


Входные данные: В файле input.txt по строкам:
размерность матрицы;
квадратная матрица;
вектор b (уравнения Ax = b);
приближённое решение x_0
коэффициент альфа;
точность;
максимальное количество итераций.
Допускается несколько тестов в одном файле.


Выходные данные: Программа сначала проходит первый тест всеми способам, потом второй и т.д. Итоги тестов выводит в файле output.txt, где для каждого теста выводится количество итераций, найденное решение и точность (в данном случае под точностью подразумевается невязка, то есть евклидова норма величины Ax_{найденное} - b).
Также программа генерирует с помощью gnuplot график сходимости для каждого метода логика создания названия такова:
"data*xx.png", где * — номер теста, а xx обозначают конкретный метод (1-9 — простая итерация, 10-90 — метод скорейшего спуска, 100-900 — метод Гаусса-Зейделя).


Проверка производилась: Linux Ubuntu 22.04.3 LTS AMD64 (WSL 2 for Windows 11) + gcc 11.4.0


Запуск программы:
gcc main.c fun.c -lm
./a.out


Описание тестов:
В файле input.txt:
1. ленточная матрица малого порядка (на диагоналях, параллельных главной, находятся ненулевые элементы);
2. положительно определённая матрица;
3. расходящийся случай;
4. ленточная матрица большего подрядка (4×4);
5-6. несимметричные матрица с разным коэффициентом альфа;
7. несимметричная матрица;
8. матрица без строго диагонального преобладания.
В main.c:
1. тест на производительность: создаётся ленточная матрица произвольного порядка (порядок задать можно в строке 140 файле main.c).



Отчёт по тестам:
Действительно, наибольшее количество проблем возникает при использовании метода простой итерации. Он плохо масштабируется, требует подбора коэффициента альфа для адекватной сходимости. Метод Гаусса-Зейделя во многих случаях превосходит метод скорейшего спуска.
