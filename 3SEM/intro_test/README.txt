ЦЫБУЛИН ЕГОР 208 группа (2025 год)


Условие: Напишите программу, решающую следующую задачу:
Дано 3 <= n <= 10^5 точек на плоскости, они образуют выпуклый многоугольник. Найти любую прямую, которая бы разбила многоугольник на две части одинаковой площади. Координаты точек — целые числа, по модулю не превосходящие 10^5. Ограничение по времени — 1 секунда.


Входные данные: input.txt - в первой строке число n, далее по столбцам целые координаты n точек (x,y).

*ЗАМЕЧАНИЕ*: Честно говоря, наверное, можно было бы написать алгоритм распознавания выпуклой оболочки (ну, например, смотреть находятся ли точки в одной в одной полуплоскости, ограниченной прямой, проходящей через две соседние точки), но вот как построить этот выпуклый многоугольник, когда точки идут вразброс, я не знаю. Поэтому в моей программе предполагается, что точки идут последовательно, то есть если их соединить прямыми, то мы получим выпуклый многоугольник.


Выходные данные: В консоль выводит диагностику: или сообщение об ошибке, или ответ: координаты двух точек, по которым можно построить прямую, уравнение прямой, площадь одной половинки, целевая площадь половинки и площадь изначальной фигуры.


Проверка производилась: Linux Ubuntu 22.04.3 LTS AMD64 (WSL 2 for Windows 11) + gcc 11.4.0 со следующими ключами:
-W -Wall -Wfloat-equal -Wpointer-arith -Wwrite-strings -Wcast-align \
-Wformat-security -Wmissing-format-attribute -Wformat=1 \
-Wno-long-long -Wcast-align -Winline -Werror -pedantic -pedantic-errors \
-std=gnu99 -Wstrict-prototypes -Wmissing-prototypes -Wmissing-declarations \
-Wold-style-definition -Wdeclaration-after-statement -Wbad-function-cast -Wnested-externs \
-Wunused -Wuninitialized -fPIC


Запуск программы:
gcc 1.c -lm
./a.out
