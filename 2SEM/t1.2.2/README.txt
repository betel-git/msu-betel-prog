ЦЫБУЛИН ЕГОР 108 группа (2025 год)


Условие: Вычислить интеграл от данной функции на данном отрезке с заданной точностью по составной формуле ТРАПЕЦИЙ.
Входные данные: В INPUT.txt значения eps — точность вычислений, a — левая граница отрезка, b — правая граница отрезка по строкам.
Выходные данные: В консоль программа должна выводить сообщение "The results are saved in OUTPUT.txt" или сообщения об ошибке открытия файлов. В OUTPUT.txt хранятся входные значения, номер теста, предупреждения, результат работы функции, количество итераций.
Проверка производилась: Linux Ubuntu 22.04.3 LTS AMD64 (WSL 2 for Windows 11) + gcc 11.4.0


Замечание: Ограничение на количество итераций стоит на 25, потому что при большем значении ограничителя каждое новое вычисления приближённого значения интеграла будет занимать много времени, отчего всё тестирование может занять несколько минут.

Замечание: Чтобы не казалось, что программа зависла, я оставил вывод в консоль нынешнее количество отрезков n.


Состав проекта:
1. functions.c — файл с функциями
2. main.c — файл с функцией main
3. headers.h — заголовочный файл
4. INPUT.txt — входной файл
5. OUTPUT.txt — выходной файл
6. README.txt — инструкция

Запуск программы:
gcc functions.c main.c -lm
./a.out

Краткое описание тестов и поведения программы на них:
1. Быстроосциллирующая функция. Проверить можно на любом отрезке.
2. Особенность такой экспоненты в том, что она очень быстро стремится к нулю. Для неё интересен тест, где промежутки a = 3, b = 6, например. К сожалению, интеграл не берётся, поэтому я не могу посчитать точный ответ.
3. Особенность такого гиперболического тангенса в том, что он очень быстро стремится к 1 при положительных значениях x. Для него интересен тест, где промежутки a = 0, b = 0.02, например, а также a = 3, b = 6 (проверяем как ведёт себя при почти вертикальном графике в первом случае и как ведёт при почти горизонтальном графике во втором случае (впрочем, как и в прошлой функции)).
4. Пикоподобная функция. Все прошлые тесты интересны и для этой функции. Скорее всего, понадобится много времени на вычисление значения, и программа выдаст предупреждение об ограничении по количеству итераций.
5. Периодическая функция с шумами из осциллирующей функции. Аналогично с прошлой функцией требует большой точности, поэтому, скорее всего, выдаст предупреждение об ограничении по количеству итераций.
6. Такая дробь интересна поведением вблизи x = 1. Для неё существует проверка при a = 0.95, b = 0.9998.
7. Ещё одна быстроосциллирующая функция вблизи нуля. Интересна проверка при a = 0, b = 3. Остальные проверки, как можно видеть, данная функция завалила.
